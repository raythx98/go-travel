## Design Decisions

1. **finalize_itinerary tool** — structured output achieved by forcing Claude to call this tool; avoids freeform markdown parsing.
2. **ask_user context injection** — response channel and itinerary channel injected via context values to avoid circular imports between agent/server/tools.
3. **ToolInputSchemaParam Properties** — passed as `map[string]any` directly; SDK accepts `any`.
4. **Ollama tool result translation** — tool results folded into user message content string (Ollama doesn't have a native tool_result block type in /api/chat).

## Surprising Discovery

The anthropic-sdk-go v1.43.0 `Tools` field in `MessageNewParams` is `[]ToolUnionParam`, not `[]ToolParam`. Custom client tools must be wrapped: `ToolUnionParam{OfTool: &ToolParam{...}}`.
